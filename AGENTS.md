# Rules for Agents

- **GOLDEN RULE -- never compile, always download a prebuilt binary**: No package may be built from source. If a package would compile (Rust/C/etc.) because nixpkgs has no cached aarch64-darwin binary, do NOT accept the source build -- source it as a prebuilt instead: a Homebrew cask/formula (`nix/darwin/default.nix`), a mise binary backend (`aqua:`/`npm:`/`pipx:` in `nix/home/default.nix`), or a nix derivation that *fetches* the upstream release binary. Verify with `nix build .#darwinConfigurations.ranolp-work-MBP-26.system --dry-run`: if any real package (not a home-manager wrapper/activation/stub derivation) appears under "will be built", it is compiling -- stop and re-source it. Example: espanso is source-only in nixpkgs on darwin (and its Homebrew cask's nested-dmg unpack is broken under nix-homebrew), so `nix/home/darwin/default.nix` fetches the official release and mounts+`ditto`s the notarized app into `~/Applications` at activation -- download only, and unlike a `7zz` unpack it preserves the code signature (a re-packed bundle triggers "Espanso is damaged"; `codesign` can't re-seal in the nix sandbox).
- **Rebuild after dotfiles edits**: After modifying any file under `nix/home/configs/`, run `sudo darwin-rebuild switch --flake ~/.dotfiles/nix#ranolp-work-MBP-26`. Changes are not applied until rebuild completes.
- **Direct main git workflow allowed here**: When requested, agents should create commits on `main` and push `main` to `origin` in this repository. Inspect status/diff first, stage explicit paths, and keep PR workflows out of scope.
- **Never edit global config directly**: Never edit `~/.claude/settings.json` or any file under `~/.claude/` directly. All Claude config lives in `nix/home/configs/claude/` in this repo -- edit there instead.
